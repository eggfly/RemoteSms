1. 作用

在接入小米推送Android版SDK时，需要在AndroidManifest.xml中添加一些参数。为了简化修改此文件，特做此工具。

她可以实现检查现有AndroidManifest.xml参数是否齐全，或修复一些参数。

2. 使用

  1. 工具是以jar的形式提供。把此工具设置到环境变量里

  2. 进入AndroidManifest.xml所在目录

  3. 执行下面命令

    java -jar check_manifest.jar 检查现有文件是否存在问题

    java -jar check_manifest.jar -edit 修复文件

3. 检查规则

匹配当前目录下AndroidManifest.xml文件是否存在以下特性。 

 <?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
    <uses-permission android:name="android.permission.INTERNET" />
    <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
    <uses-permission android:name="android.permission.ACCESS_WIFI_STATE" />
    <uses-permission android:name="android.permission.READ_PHONE_STATE" />
    <uses-permission android:name="android.permission.GET_TASKS" />
    <uses-permission android:name="android.permission.VIBRATE"/>
    <uses-permission android:name="com.xiaomi.mipushdemo.permission.MIPUSH_RECEIVE" />
    
    <permission android:name="com.xiaomi.mipushdemo.permission.MIPUSH_RECEIVE" />
    
    <application>
        <service
            android:name="com.xiaomi.mipush.sdk.MessageHandleService"/>
        <service
            android:name="com.xiaomi.push.service.XMPushService"/>
        <service
            android:exported="true" 
            android:name="com.xiaomi.mipush.sdk.PushMessageHandler" />
        
        <receiver
            android:exported="true"
            android:name="com.xiaomi.push.service.receivers.NetworkStatusReceiver" >
            <intent-filter>
                <action android:name="android.net.conn.CONNECTIVITY_CHANGE" />
            </intent-filter>
        </receiver>
        <receiver
            android:exported="false"
            android:name="com.xiaomi.push.service.receivers.PingReceiver" >
            <intent-filter>
                <action android:name="com.xiaomi.push.PING_TIMER" />
            </intent-filter>
        </receiver>
    </application>
</manifest>
